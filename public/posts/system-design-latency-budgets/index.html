System Design Notebook: Latency Budgets

2026-02-14

When evaluating distributed systems, I start by writing down a latency budget before discussing architecture.

  • network round trips
  • storage access time
  • serialization overhead
  • retry behavior under partial failure

If we cannot explain where every millisecond goes, we are not really doing system design yet.
